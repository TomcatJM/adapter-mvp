## 2. 各文档职责

| 文档 | 用途 |
| --- | --- |
| `README.md` | 项目入口和快速导航 |
| `./使用手册.md` | 日常使用、联调用法、主流程操作 |
| `../templates/钉钉需求文档标准模版.md` | 产品编写需求、AI 解析前对齐的标准模版 |
| `./钉钉需求文档解析临时指导.md` | 钉钉需求拆分、需求/任务层级、字段映射口径 |
| `./架构设计.md` | 系统分层、状态机、绑定规则 |
| `./部署与配置.md` | 本地/远端部署、环境变量、配置脚本 |
| `./当前进度说明.md` | 当前完成度和下一步 |
| `./主流程落地方案.md` | 总体方案和取舍依据 |
| `./后续配置清单.md` | 还需要补齐的配置 |
| `./提交说明模板.md` | 提交说明规范 |

## 3. 建议维护方式

- 产品文档格式变化时，先更新 `../templates/钉钉需求文档标准模版.md`，再调整解析和云效创建逻辑。
- 新能力先写进 `主流程落地方案` 或 `后续配置清单`，再改代码。
- 需要操作者直接上手的流程，优先写进 `使用手册`。
- 涉及表结构、状态机、绑定规则的改动，优先同步到 `架构设计`。
- 涉及部署、环境变量、密钥和脚本的改动，优先同步到 `部署与配置`。
