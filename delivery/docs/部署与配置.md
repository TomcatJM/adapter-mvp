# Adapter MVP 部署与配置

## 0. 初始化需要什么

如果只是把项目拉到远端，先分成两层看：

### 0.1 最低可启动

只要先把服务跑起来，最低需要：

| 配置 | 作用 |
| --- | --- |
| `ADAPTER_API_TOKEN` | 接口鉴权 |
| `ADAPTER_DB_HOST` / `ADAPTER_DB_PORT` / `ADAPTER_DB_NAME` / `ADAPTER_DB_USER` / `ADAPTER_DB_PASSWORD` | MySQL 连接 |
| `ADAPTER_PUBLIC_BASE_URL` | 对外地址，便于回调和调试 |

这层只保证 HTTP 服务能启动，不代表钉钉、云效、Apifox 已经打通。

### 0.2 完整链路必配

要跑完整闭环，还要补下面这些：

| 账号 / 模块 | 需要补的东西 | 中文说明 | 主要存放位置 |
| --- | --- | --- | --- |
| Adapter 服务账号 | `ADAPTER_API_TOKEN` | Adapter 自身接口的鉴权令牌 | 环境变量 / 远端 env |
| Adapter 数据库账号 | `ADAPTER_DB_HOST` / `ADAPTER_DB_PORT` / `ADAPTER_DB_NAME` / `ADAPTER_DB_USER` / `ADAPTER_DB_PASSWORD` | Adapter 访问 MySQL 的账号 | 环境变量 / 远端 env |
| 远端执行开关 | `ALLOW_REMOTE_EXEC=true` | 是否允许真实 SSH 执行 | 远端 env |
| SSH 目标主机账号 | `config/hosts.masked.json`、`secrets/hosts.secrets.json` | 目标服务器登录账号和密码引用 | 配置文件 / secrets |
| 钉钉应用账号 | `DINGTALK_APP_NAME` / `DINGTALK_APP_KEY` / `DINGTALK_APP_SECRET` / `DINGTALK_OPERATOR_ID` | 读取钉钉 / Alidocs 文档的应用账号 | `adapter_dingtalk_app`、`adapter_dingtalk_doc_config` |
| 云效集成账号 | `account_name` / `auth_type` / `access_key_id` / `access_key_secret` / `legacy_token` / `security_token` | 调云效 OpenAPI 的主账号 | `adapter_yunxiao_account_config` |
| 云效项目账号 | `organization_id` / `project_id` / `workitem_type_identifier` / `task_workitem_type_identifier` / `done_status_id` / `close_transition_id` | 某个业务项目对应的云效项目配置；需求和任务是两类工作项，必须分别配置类型 ID | `adapter_yunxiao_project_config` |
| 云效人员账号 | `member_name` / `yunxiao_account_id` / `enabled` | 云效里负责创建和关单的人 | `adapter_yunxiao_member` |
| 云效项目负责人映射 | `project_name` / `yunxiao_account_id` / `is_default` / `enabled` | 某项目默认负责人和候选负责人 | `adapter_yunxiao_project_member_relation` |
| Apifox 项目账号 | `APIFOX_ACCESS_TOKEN` / `project_name` / `project_id` | 把 OpenAPI 导入到哪个 Apifox 项目 | `adapter_apifox_project_config` 或环境变量 |
| Apifox 流水线映射 | `pipeline_id -> project_name` | 哪条云效流水线对应哪个 Apifox 项目 | `adapter_apifox_pipeline_config` |

如果你只想先验证“服务能不能起来”，先配 0.1。
如果你要验证“钉钉 -> 云效 -> 流水线 -> Apifox -> 关单”这条链路，0.2 也要补齐。

```powershell

```

## 2. 远端部署

当前推荐的远端路径：

```text
应用目录：/opt/adapter-mvp
服务名：adapter-mvp
监听端口：18080
```

历史上已验证可用的部署流程是：

```bash
git archive --format=tar.gz -o adapter-mvp.tar.gz HEAD
scp adapter-mvp.tar.gz root@47.116.102.238:/opt/adapter-mvp/adapter-mvp.tar.gz
ssh root@47.116.102.238 'cd /opt/adapter-mvp && tar -xzf adapter-mvp.tar.gz && APP_DIR=/opt/adapter-mvp PORT=18080 bash scripts/remote_install.sh'
```

`scripts/remote_install.sh` 会负责创建虚拟环境、安装依赖、写 systemd service、写 logrotate 配置并重启服务。

## 3. 核心环境变量

| 变量 | 作用 |
| --- | --- |
| `ADAPTER_API_TOKEN` | 业务接口鉴权 token |
| `ADAPTER_DB_HOST` / `ADAPTER_DB_PORT` / `ADAPTER_DB_NAME` / `ADAPTER_DB_USER` / `ADAPTER_DB_PASSWORD` | MySQL 连接 |
| `ADAPTER_PUBLIC_BASE_URL` | 对外基础地址 |
| `ALLOW_REMOTE_EXEC` | 是否允许真实远端执行 |

## 4. 云效配置

建议通过 `scripts/upsert_yunxiao_config.py` 写库。

### 4.1 账号配置

| 变量 | 作用 |
| --- | --- |
| `auth_type` | `legacy_token`、`acs_ak` 或 `personal_token` |
| `access_key_id` / `access_key_secret` | 云效 OpenAPI 鉴权 |
| `legacy_token` | 兼容创建链路 |
| `organization_id` / `project_id` | 云效组织和项目 |
| `workitem_type_identifier` | 云效“需求”工作项类型 ID |
| `task_workitem_type_identifier` | 云效“任务”工作项类型 ID；钉钉需求拆出任务时必填 |
| `sprint_id` | 可选固定迭代 ID；如果为空且需求文档有版本号，personal token 链路会按版本号匹配云效迭代并写入创建 payload 的 `sprint` 字段 |
| `done_status_id` / `close_transition_id` | 关单目标 |

### 4.2 人员配置

- 人员表：姓名、云效账号 ID、是否启用
- 项目人员关联表：项目、人员、默认负责人标记、是否启用

如果需求里明确写了负责人，就按负责人匹配；没有指定时，使用项目默认负责人。

## 5. 钉钉配置

建议通过 `scripts/upsert_dingtalk_config.py` 维护。

必配内容：

- `DINGTALK_APP_NAME`
- `DINGTALK_APP_KEY`
- `DINGTALK_APP_SECRET`
- `DINGTALK_OPERATOR_ID`

## 6. Apifox 配置

建议通过 `scripts/upsert_apifox_project_config.py` 和 `scripts/upsert_apifox_pipeline_config.py` 维护。

常见变量：

- `APIFOX_AUTO_IMPORT`
- `APIFOX_ACCESS_TOKEN`
- `APIFOX_PROJECT_<KEY>_ID`
- `APIFOX_PIPELINE_<PIPELINE_ID>_PROJECT`
- `OPENAPI_URL`
- `APIFOX_OPENAPI_URL_TEMPLATE`

## 8. 常见问题

- 钉钉读不到：先看 app credential 和 doc config。
- 云效任务创建失败：先看 project config、人员表、账号鉴权。
- 关单失败：优先检查 `done_status_id`、`close_transition_id` 和真实云效账号权限。
- 不要依赖静默默认项目 ID；缺配置时应该明确报错。
