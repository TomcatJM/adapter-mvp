# Adapter MVP 部署与配置

## 0. 初始化需要什么

如果只是把项目拉到远端，先分成两层看：

### 0.1 最低可启动

只要先把服务跑起来，最低需要：

| 配置 | 作用 |
| --- | --- |
| `adapter_api_client` 或 `ADAPTER_API_TOKEN` | Adapter 业务接口鉴权；推荐用数据库表维护多调用方 token，环境变量仅作兼容兜底 |
| `ADAPTER_DB_HOST` / `ADAPTER_DB_PORT` / `ADAPTER_DB_NAME` / `ADAPTER_DB_USER` / `ADAPTER_DB_PASSWORD` | MySQL 连接 |
| `ADAPTER_PUBLIC_BASE_URL` | 对外地址，便于回调和调试 |

这层只保证 HTTP 服务能启动，不代表钉钉、云效、Apifox 已经打通。

### 0.2 完整链路必配

要跑完整闭环，还要补下面这些：

| 账号 / 模块 | 需要补的东西 | 中文说明 | 主要存放位置 |
| --- | --- | --- | --- |
| Adapter 服务账号 | `adapter_api_client.client_id` / `token_hash` / `scopes`，兼容 `ADAPTER_API_TOKEN` | Adapter 自身接口的调用方鉴权；数据库只保存 token 哈希，支持多用户/多系统 | `adapter_api_client` / 环境变量兜底 |
| Adapter 数据库账号 | `ADAPTER_DB_HOST` / `ADAPTER_DB_PORT` / `ADAPTER_DB_NAME` / `ADAPTER_DB_USER` / `ADAPTER_DB_PASSWORD` | Adapter 访问 MySQL 的账号 | 环境变量 / 远端 env |
| 远端执行开关 | `ALLOW_REMOTE_EXEC=true` | 是否允许真实 SSH 执行 | 远端 env |
| SSH 目标主机账号 | `config/hosts.masked.json`、`secrets/hosts.secrets.json` | 目标服务器登录账号和密码引用 | 配置文件 / secrets |
| 钉钉应用账号 | `DINGTALK_APP_NAME` / `DINGTALK_APP_KEY` / `DINGTALK_APP_SECRET` / `DINGTALK_OPERATOR_ID` | 读取钉钉 / Alidocs 文档的应用账号 | `adapter_dingtalk_app`、`adapter_dingtalk_doc_config` |
| 云效集成账号 | `account_name` / `auth_type` / `access_key_id` / `access_key_secret` / `legacy_token` / `security_token` | 调云效 OpenAPI 的主账号 | `adapter_yunxiao_account_config` |
| 云效项目账号 | `organization_id` / `project_id` / `workitem_type_identifier` / `task_workitem_type_identifier` / `done_status_id` / `close_transition_id` | 某个业务项目对应的云效项目配置；需求和任务是两类工作项，必须分别配置类型 ID | `adapter_yunxiao_project_config` |
| 云效人员账号 | `member_name` / `yunxiao_account_id` / `enabled` | 云效里负责创建和关单的人 | `adapter_yunxiao_member` |
| 云效项目负责人映射 | `project_name` / `yunxiao_account_id` / `is_default` / `enabled` | 某项目默认负责人和候选负责人 | `adapter_yunxiao_project_member_relation` |
| Apifox 账号 Token | `account_name` / `access_token` / `enabled` | 调 Apifox OpenAPI 导入接口的账号凭证 | `adapter_apifox_account_config`，环境变量 `APIFOX_ACCESS_TOKEN` 仅作兜底 |
| Apifox 项目账号 | `account_name` / `project_name` / `project_id` | 把 OpenAPI 导入到哪个 Apifox 项目，并绑定使用哪个 Apifox 账号 | `adapter_apifox_project_config` 或环境变量 |
| Apifox 流水线映射 | `pipeline_id` / `pipeline_name` / `service_name` / `env_name` / `repo_name` / `apifox_project_config_id` | 哪条云效流水线对应哪个服务和 Apifox 项目；不再通过项目名间接关联 | `adapter_apifox_pipeline_config` |

说明：业务字段仍作为外部系统入口和人工配置入口保留，例如 Apifox 项目自身的 `project_name`、云效项目自身的 `project_name`、`account_name`、`yunxiao_account_id`、`pipeline_id`。内部表关系会同步维护主键 ID 字段，例如 `account_config_id`、`project_config_id`、`member_id`、`apifox_project_config_id`，避免改名或大小写差异导致关系断裂。流水线映射表不再保存 `project_name`，避免用名称做跨表关联。

如果你只想先验证“服务能不能起来”，先配 0.1。
如果你要验证“钉钉 -> 云效 -> 流水线 -> Apifox -> 关单”这条链路，0.2 也要补齐。

## 2. 远端部署

当前推荐的远端路径：

```text
应用目录：/opt/adapter-mvp
服务名：adapter-mvp
监听端口：18080
```

历史上已验证可用的部署流程是：

```bash
git archive --format=tar.gz -o adapter-mvp.tar.gz HEAD
scp adapter-mvp.tar.gz root@47.116.102.238:/opt/adapter-mvp/adapter-mvp.tar.gz
ssh root@47.116.102.238 'cd /opt/adapter-mvp && tar -xzf adapter-mvp.tar.gz && APP_DIR=/opt/adapter-mvp PORT=18080 bash scripts/remote_install.sh'
```

`scripts/remote_install.sh` 会负责创建虚拟环境、安装依赖、写 systemd service、写 logrotate 配置并重启服务。

## 3. 核心环境变量

| 变量 | 作用 |
| --- | --- |
| `ADAPTER_API_TOKEN` | 业务接口鉴权 token，兼容兜底；新调用方优先写入 `adapter_api_client` |
| `ADAPTER_DB_HOST` / `ADAPTER_DB_PORT` / `ADAPTER_DB_NAME` / `ADAPTER_DB_USER` / `ADAPTER_DB_PASSWORD` | MySQL 连接 |
| `ADAPTER_PUBLIC_BASE_URL` | 对外基础地址 |
| `ALLOW_REMOTE_EXEC` | 是否允许真实远端执行 |

### 3.1 Adapter API 调用方 Token

推荐用 `adapter_api_client` 表维护调用方，不再只依赖单个 `ADAPTER_API_TOKEN`。

| 字段 | 说明 |
| --- | --- |
| `client_id` | 调用方唯一标识，例如 `codex-local`、`yunxiao-flow` |
| `client_name` | 调用方中文名称 |
| `token_hash` | API token 的 SHA-256 哈希，用于鉴权比对 |
| `token_plaintext` | API token 原始明文，用于防误删恢复；禁止打印到日志、提交到代码或粘贴到聊天 |
| `scopes` | 权限范围，逗号分隔；普通 workflow 可用 `workflow:read,workflow:write`，删除云效工作项必须额外具备 `yunxiao:delete` |
| `enabled` | 是否启用 |
| `expires_at` | 过期时间，空表示不过期 |

写入示例：

```bash
ADAPTER_CLIENT_TOKEN='<不要写入文档或聊天>' \
python scripts/upsert_api_client.py \
  --client-id adapter-default \
  --client-name Adapter默认调用方 \
  --scopes workflow:read,workflow:write,yunxiao:delete \
  --created-by system
```

默认调用方按管理员使用，给最大权限。删除云效需求/任务属于高风险操作，不使用未入库的 `ADAPTER_API_TOKEN` 兜底；其他低权限调用方可以单独创建数据库 token，只授予需要的 scope。

单独创建删除管理员 token 示例：

```bash
ADAPTER_CLIENT_TOKEN='<不要写入文档或聊天>' \
python scripts/upsert_api_client.py \
  --client-id yunxiao-delete-admin \
  --client-name 云效删除管理员 \
  --scopes workflow:read,workflow:write,yunxiao:delete \
  --created-by 姬志猛
```

调用时仍使用标准请求头：

```http
Authorization: Bearer <调用方自己的明文 token>
```

## 4. 云效配置

建议通过 `scripts/upsert_yunxiao_config.py` 写库。

### 4.1 账号配置

| 变量 | 作用 |
| --- | --- |
| `auth_type` | `legacy_token`、`acs_ak` 或 `personal_token` |
| `access_key_id` / `access_key_secret` | 云效 OpenAPI 鉴权 |
| `legacy_token` | 兼容创建链路 |
| `organization_id` / `project_id` | 云效组织和项目 |
| `workitem_type_identifier` | 云效“需求”工作项类型 ID |
| `task_workitem_type_identifier` | 云效“任务”工作项类型 ID；钉钉需求拆出任务时必填 |
| `sprint_id` | 可选固定迭代 ID；如果为空且需求文档有版本号，personal token 链路会按版本号匹配云效迭代并写入创建 payload 的 `sprint` 字段 |
| `done_status_id` / `close_transition_id` | 关单目标 |

### 4.2 人员配置

- 人员表：姓名、云效账号 ID、是否启用
- 项目人员关联表：项目、人员、默认负责人标记、是否启用

如果需求里明确写了负责人，就按负责人匹配；没有指定时，使用项目默认负责人。

## 5. 钉钉配置

建议通过 `scripts/upsert_dingtalk_config.py` 维护。

必配内容：

- `DINGTALK_APP_NAME`
- `DINGTALK_APP_KEY`
- `DINGTALK_APP_SECRET`
- `DINGTALK_OPERATOR_ID`

## 6. Apifox 配置

建议通过 `scripts/upsert_apifox_account_config.py`、`scripts/upsert_apifox_project_config.py` 和 `scripts/upsert_apifox_pipeline_config.py` 维护。

常见变量：

- `APIFOX_AUTO_IMPORT`
- `APIFOX_ACCESS_TOKEN`：Apifox 导入 token 兜底；项目已绑定 `adapter_apifox_account_config` 时优先使用 DB token
- `APIFOX_PROJECT_<KEY>_ID`
- `OPENAPI_URL`
- `APIFOX_OPENAPI_URL_TEMPLATE`

## 8. 常见问题

- 钉钉读不到：先看 app credential 和 doc config。
- 云效任务创建失败：先看 project config、人员表、账号鉴权。
- 关单失败：优先检查 `done_status_id`、`close_transition_id` 和真实云效账号权限。
- 不要依赖静默默认项目 ID；缺配置时应该明确报错。
