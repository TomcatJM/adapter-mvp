# 标准提交说明模板

## 1. 推荐模板

```text
<type>: <一句话说明>

需求来源: <钉钉文档标题或链接>
云效任务: <云效任务展示ID，多个用顿号或逗号分隔，例如 VEGZ-1186、VEGZ-1187>

变更内容:
- <变更点1>
- <变更点2>

验证:
- <测试项1>
- <测试项2>
```

## 2. 英文字段模板

如果提交工具或流水线更适合解析英文字段，使用下面模板：

```text
<type>: <summary>

DINGTALK_DOC=<document title or url>
YUNXIAO_TASK_DISPLAY_ID=<云效任务展示ID，多个用英文逗号分隔，例如 VEGZ-1186,VEGZ-1187>

Changes:
- <change 1>
- <change 2>

Verification:
- <test 1>
- <test 2>
```

## 3. 示例

```text
feat: 创建学生信息并创建跟进记录

需求来源: 校务2-1.adoc
云效任务: VEGZ-1186、VEGZ-1187

变更内容:
- 新增学生信息创建时姓名、手机号必填校验
- 新增跟进记录创建时内容必填校验

验证:
- 姓名为空返回校验错误
- 手机号为空返回校验错误
- 跟进内容为空返回校验错误
- 学生信息和跟进记录正常创建链路通过
```

GitLab 提交说明也可以直接写成这样，Webhook 会从这次提交的 commit message 里读取：

```text
feat: 修复学生列表筛选

云效任务: VEGZ-1186
```

## 4. 约定

- `云效任务` 是后续关单白名单字段；关单只关闭这里显式列出的云效任务。
- `云效任务` 支持多个 ID，例如 `云效任务: VEGZ-1186、VEGZ-1187、VEGZ-1188`。
- 如果提交说明没有 `云效任务` ID，Adapter 不执行云效关单。
- 如果流水线没有带到提交说明，手动调用 `/workflow/{id}/advance` 时可用 `closeTaskRefs` 补传同一批任务展示 ID。
- `WORKFLOW_ID` / `工作流ID` 仍可用于绑定 workflow，但不能替代 `云效任务` 关单白名单。
- `YUNXIAO_TASK_DISPLAY_ID` / `yunxiaoTaskDisplayId` / `云效展示ID` 继续兼容，填云效页面左上角展示的 ID，例如 `VEGZ-1186`。
- Webhook 读取的是本次提交的 `commitMessage`，不是分支最新提交。
- 提交解析不会卡死在固定字段名；`云效id`、`云效任务`、`云效任务编号` 等中文 key 也兼容。
- `YUNXIAO_TASK_ID` / `云效任务ID` 继续兼容；历史提交里也可以填页面展示 ID。
- Adapter 内部仍保存 URL 中的 `workitemIdentifier` 用于评论和关单；Webhook 绑定同时兼容内部 ID 和页面展示 ID。
- CI/CD webhook 必须使用本次构建的 `commitId` 对应提交说明，不要读取分支最新提交。
- 如果提交说明存在任务字段但 Adapter 查不到对应 workflow，不继续使用项目兜底绑定，避免串错任务。
- 如果提交说明没有任务字段，Adapter 可以继续使用 `branch+commit`、`pipelineId+buildNumber`、项目唯一活跃 workflow 等兜底方式绑定 workflow，但绑定成功后也不做云效关单。
