# Adapter MVP 后续配置清单

> 这份文档只收纳后续还需要补齐、或者换机器时容易漏掉的配置。当前已经能跑通的配置只在这里做归纳，不再散落到别的文档里。

## 1. 先看结论

- 当前云效创建联调用的是 `legacy_token`，所以 `access_key_id` / `access_key_secret` 为空是正常的。
- 云效关闭/回写已经验证不能继续使用 `legacy_token`：旧 `openapi-rdc.aliyuncs.com` 对工作项详情/评论/状态接口返回跳转到帮助文档，不是可用业务 API。
- 因此真实关单闭环必须切到云效官方 `acs_ak` 路径，这两项必须补齐。
- 远端已核对：`/etc/adapter-mvp/adapter-mvp.env` 里没有 `ALIBABA_CLOUD_*` / `ALIYUN_*` / `YUNXIAO_*` 账号变量，`/root/.aliyun/config.json` 虽有默认 profile 但 AK/Secret 为空，暂不能复用。
- 真正后续要持续维护的，是云效项目/人员映射、云效关单闭环参数、Apifox 项目/流水线映射、钉钉读取配置。

## 3. 当前优先级

1. 新业务项目接入时，先补云效项目映射、人员表、关单完成状态或关闭流转。
2. 新 Apifox 项目或新流水线接入时，再补 Apifox 的项目/流水线映射；如果项目 OpenAPI 不在统一网关上，同时补 `openapi_url`。
3. 真实关单联调前，必须先补 `acs_ak` 的 AK / Secret，并把项目映射切到该账号。
4. 当前 `done_status_id=100014` 已确认为“已完成”，可先用 `done_status_id` 路径联调；如果云效项目要求流程流转，再补 `close_transition_id`。

## 4. 维护规则

- 新增配置先写进这里，再改实现。
- 任何密钥只放在环境变量、远端 env 或本地 `secrets/`，不要写进文档正文。
- 如果某项配置已经由 DB 维护，就不要再额外依赖“默认项目 ID”一类的静默兜底。
