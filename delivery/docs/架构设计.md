# Adapter MVP 架构设计

## 1. 目标

`adapter-mvp` 的目标不是“代替人写代码”，而是把需求、任务、流水线、接口同步和关单串成一条可追踪的交付链。

## 2. 核心组件

| 组件 | 作用 |
| --- | --- |
| `app/main.py` | HTTP 入口，承接 adapter、workflow、云效回调和钉钉读取接口 |
| `app/workflow.py` | workflow 状态机，负责推进主链路 |
| `app/dingtalk_docs.py` | 钉钉/Alidocs 读取适配 |
| `app/yunxiao.py` | 云效工作项创建、回写、关单 |
| `app/yunxiao_pipeline.py` | 流水线回调绑定、Apifox 导入、失败处理 |
| `app/apifox.py` | OpenAPI 导入和项目映射 |
| `app/db.py` | 持久化、配置表、workflow 账本 |
| `app/audit.py` | 审计记录 |
| `app/status_store.py` | 运行状态缓存 |

## 3. 主流程

```mermaid
flowchart LR
    A["钉钉需求文档"] --> B["workflow/start"]
    B --> C["CREATED"]
    C --> D["读取文档"]
    D --> E["DOC_READ"]
    E --> F["提交结构化需求"]
    F --> G["REQUIREMENT_PARSED"]
    G --> H["创建云效工作项"]
    H --> I["YUNXIAO_TASK_CREATED / CODING_REQUESTED"]
    I --> J["代码提交 + 流水线回调"]
    J --> K["PIPELINE_SUCCESS"]
    K --> L["Apifox 导入"]
    L --> M["APIFOX_SYNCED"]
    M --> N["关闭云效任务"]
    N --> O["YUNXIAO_TASK_CLOSED"]
```

## 4. 状态机

主链路当前重点状态如下：

| 状态 | 含义 |
| --- | --- |
| `CREATED` | workflow 已创建，等待读取文档 |
| `DOC_READ` | 钉钉文档已读入 workflow |
| `REQUIREMENT_PARSED` | 需求已结构化，等待创建云效任务 |
| `YUNXIAO_TASK_CREATED` | 云效工作项已创建 |
| `CODING_REQUESTED` | 等待或正在编码 |
| `PIPELINE_RUNNING` | 流水线运行中 |
| `PIPELINE_SUCCESS` | 流水线成功 |
| `APIFOX_SYNCED` | Apifox 同步完成 |
| `YUNXIAO_TASK_CLOSED` | 云效任务已关闭 |
| `PIPELINE_FAILED` | 流水线失败 |
| `NEEDS_HUMAN` | 需要人工处理 |

## 5. 绑定规则

流水线回调绑定 workflow 的优先级是：

1. `WORKFLOW_ID`
2. 云效工作项 ID / 提交说明里的展示 ID
3. `pipelineId + buildNumber`
4. `branchName + commitId`
5. 同项目唯一活跃 workflow 兜底

这里最重要的一点是：回调读取的是本次构建对应的提交信息，不是分支当前最新提交。

## 6. 数据层

关键表大致分成三组：

- workflow 账本：`adapter_workflow_instance`、`adapter_workflow_event`
- 云效配置：`adapter_yunxiao_account_config`、`adapter_yunxiao_project_config`、`adapter_yunxiao_member`、`adapter_yunxiao_project_member_relation`
- 读取和同步配置：`adapter_dingtalk_app`、`adapter_dingtalk_doc_config`、`adapter_apifox_account_config`、`adapter_apifox_project_config`、`adapter_apifox_pipeline_config`

## 7. 设计约束

- 不做静默默认项目兜底。
- 不把密钥写进日志、提交说明或文档。
- workflow 是主事实源，审计只是证据，不是状态机。
- 云效、Apifox、钉钉都通过配置表或环境变量显式绑定。
