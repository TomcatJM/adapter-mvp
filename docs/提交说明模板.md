# 标准提交说明模板

## 1. 推荐模板

```text
<type>: <一句话说明>

需求来源: <钉钉文档标题或链接>
云效任务ID: <云效页面展示ID，例如 VEGZ-1186>
工作流ID: <workflowId>

变更内容:
- <变更点1>
- <变更点2>

验证:
- <测试项1>
- <测试项2>
```

## 2. 英文字段模板

如果提交工具或流水线更适合解析英文字段，使用下面模板：

```text
<type>: <summary>

DINGTALK_DOC=<document title or url>
YUNXIAO_TASK_ID=<云效页面展示ID，例如 VEGZ-1186>
WORKFLOW_ID=<workflowId>

Changes:
- <change 1>
- <change 2>

Verification:
- <test 1>
- <test 2>
```

## 3. 示例

```text
feat: 创建学生信息并创建跟进记录

需求来源: 校务2-1.adoc
云效任务ID: VEGZ-1186
工作流ID: wf-27f5462a4ec047f1

变更内容:
- 新增学生信息创建时姓名、手机号必填校验
- 新增跟进记录创建时内容必填校验

验证:
- 姓名为空返回校验错误
- 手机号为空返回校验错误
- 跟进内容为空返回校验错误
- 学生信息和跟进记录正常创建链路通过
```

## 4. 约定

- `WORKFLOW_ID` / `工作流ID` 是最高优先级绑定字段。
- `YUNXIAO_TASK_ID` / `云效任务ID` 是第二优先级绑定字段，优先填云效页面左上角展示的 ID，例如 `VEGZ-1186`。
- Adapter 内部仍保存 URL 中的 `workitemIdentifier` 用于评论和关单；Webhook 绑定同时兼容内部 ID 和页面展示 ID。
- CI/CD webhook 必须使用本次构建的 `commitId` 对应提交说明，不要读取分支最新提交。
- 如果提交说明存在任务字段但 Adapter 查不到对应 workflow，不继续使用项目兜底绑定，避免串错任务。
- 如果提交说明没有任务字段，Adapter 才继续使用 `branch+commit`、`pipelineId+buildNumber`、项目唯一活跃 workflow 等兜底策略。
