# Adapter MVP 后续配置清单

> 这份文档只收纳后续还需要补齐、或者换机器时容易漏掉的配置。当前已经能跑通的配置只在这里做归纳，不再散落到别的文档里。

## 1. 先看结论

- 当前云效联调用的是 `legacy_token`，所以 `access_key_id` / `access_key_secret` 为空是正常的。
- 如果后面切到云效官方 `acs_ak` 路径，这两项必须补齐。
- 真正后续要持续维护的，是云效项目/人员映射、Apifox 项目/流水线映射、钉钉读取配置、以及云效关单闭环参数。

## 3. 当前优先级

1. 新业务项目接入时，先补云效项目映射和人员表。
2. 新 Apifox 项目或新流水线接入时，再补 Apifox 的项目/流水线映射。
3. 切换云效正式 OpenAPI 时，再补 AK / Secret。
4. 云效关单闭环实现时，再补关单状态字段。

## 4. 维护规则

- 新增配置先写进这里，再改实现。
- 任何密钥只放在环境变量、远端 env 或本地 `secrets/`，不要写进文档正文。
- 如果某项配置已经由 DB 维护，就不要再额外依赖“默认项目 ID”一类的静默兜底。
